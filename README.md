# 🇰🇷 Korean Easy — Case Study

- **Live app:** <https://koreaneasy.com/>
- **Role:** Solo founder, designer & developer
- **Timeline:** 2026–present

## Overview

KoreanEasy is a web-based Korean language learning app built for English speakers who want to start from zero. The focus is on Hangeul (the Korean alphabet), pronunciation, vocabulary, and grammar — delivered through an interactive, mobile-first UI.

The project is a solo commercial product: one developer, one designer, one deployment pipeline. Built entirely in spare hours — evenings and weekends.

## Problem

Most Korean learning resources online are either paywalled apps optimised for engagement metrics, or static PDFs that offer no interactivity. There is a gap for a focused, distraction-free tool that teaches the Korean alphabet correctly from day one — with real pronunciation audio, stroke order, and meaningful examples.

Neither category teaches the script, vocabulary, and grammar in a way that builds genuine intuition for English speakers. KoreanEasy was built to bridge that gap.

## Features

- Complete Hangeul alphabet with stroke order animations
- Interactive letter cards — consonants, vowels, compound vowels
- English loanwords in Korean (with audio)
- Korean syllables table
- Dictionary with individual word pages and example sentences
- SRS vocabulary quiz (SM-2 spaced repetition)
- Grammar lessons
- AI and human voice audio comparison player
- Personal vocabulary list with progress tracking
- Word pairing practice
- Sentence builder
- Blog
- Newsletter + free PDF lead magnets
- GDPR-compliant consent flow

## Mascot

**Hangurang (한구랑)** is the app's original mascot — a Korean tiger scholar, the name combines:

- *han* (한, Korean)
- *gu* (구, speech)
- *rang* (랑, tiger)

## What I Learned

- Shipping a commercial product solo requires tight scope discipline — features that don't serve the core learning flow were cut or deferred
- Korean font rendering across browsers and operating systems is genuinely inconsistent; solving it with CSS custom properties and a dedicated context use
- Separating human-recorded from AI-generated audio is not just a labelling convention — it matters for learner trust and content quality signalling
